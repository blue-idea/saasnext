# 实施计划

执行时需严格遵循 `docs/spec/requirements.md` 中对应需求的验收标准。每项任务通过 `_需求:` 引用相关需求编号。

## 基础环境配置

- [ ] 1. 项目初始化与核心配置
    - 初始化 Next.js (App Router), TypeScript。
    - 配置 Drizzle ORM 与 Postgres 连接及 `drizzle.config.ts`。
    - 配置 `next-intl` 多语言路由与中间件。
    - 安装并配置 Shadcn UI 基础组件方案。
    - **Git Hooks**: 安装并配置 Husky + lint-staged，强制执行提交前检查（lint, type-check）。
    - **GitHub Actions**: 配置 CI/CD 工作流，包含代码审计、安全扫描（Gitleaks/Semgrep）与自动构建。
    - **验证**：编译通过、DB 连接成功，且测试 git commit 被 hook 拦截及 CI 流程成功运行。
    - _需求: 基础架构引用, 需求 17 (安全基线)。

## 认证与账号体系

- [ ] 2. 统一认证流实现
    - 集成 Next-Auth，支持 GitHub OAuth 与 Email/Password。
    - 完成 `ui/登录` 与 `ui/注册页` UI 编写。
    - 实现新用户注册发放 10 初始积分逻辑。
    - **验证**：成功登录后 `users` 表创建记录并伴随积分发放记录。
    - _需求: 需求 2, 需求 7.2。

- [ ] 3. 后台鉴权与白名单过滤
    - 实现 `/[locale]/admin/*` 路由的中间件级别鉴权。
    - 对比 `ADMIN_EMAILS` 环境变量（忽略大小写）。
    - 编写 `ui/后台-登录` 页面。
    - **验证**：非白名单用户访问 `/admin` 返回 403 "No access"。
    - _需求: 需求 1, 需求 2.5, 2.6, 2.7。

## 布局与主题系统

- [ ] 4. 路由隔离与响应式布局
    - 创建前台 `DefaultLayout` 与后台 `DashboardLayout`（包含 Sidebar）。
    - 编写 `ui/首页` 完整内容（产品价值、功能亮点）。
    - **验证**：`/admin` 路径展示侧边栏，其他路径不展示。
    - _需求: 需求 1, 需求 3.1。

- [ ] 5. 动态主题/皮肤系统
    - 实现基于 CSS Variables 的主题配置消费。
    - 编写 `ui/后台-皮肤管理` UI，支持配置 Logo、圆角、品牌色。
    - **验证**：后台修改圆角值后，前台组件样式同步更新。
    - _需求: 需求 14。

## 交易与积分子系统

- [ ] 6. 套餐展示与下单流程

- [ ] 7. 统一结算回调与权益发放

    - _需求: 需求 5, 需求 6, 需求 7.2。
